# دليل استخدام مشروع سباق ترتيب الدوري السوري

## المتطلبات الأساسية

قبل البدء تأكد من تثبيت:
- **Node.js** الإصدار 18 أو أحدث — [nodejs.org](https://nodejs.org)
- **npm** يأتي مع Node.js تلقائياً

---

## الخطوة 1 — تشغيل المشروع

افتح مجلد المشروع في الطرفية ونفّذ:

```bash
npm run dev
```

ثم افتح المتصفح على العنوان: **http://localhost:5173**

---

## الخطوة 2 — إضافة شعارات الأندية

1. ضع صور PNG لشعارات الأندية داخل المجلد:
   ```
   src/assets/logos/
   ```
   مثال: `wahda.png`، `jaish.png`، `tishreen.png` ...

2. يُفضّل أن تكون الصور:
   - مربعة الشكل (مثلاً 64×64 بكسل أو أكبر)
   - خلفية شفافة (PNG)

---

## الخطوة 3 — إدخال بيانات الترتيب الحقيقية

افتح الملف:
```
src/data/standings.json
```

هذا هو هيكل الملف — عدّل القيم بالبيانات الحقيقية:

```json
{
  "matchweeks": [1, 2, 3, ..., 26],
  "teams": [
    {
      "name": "الوحدة",
      "abbr": "WDH",
      "color": "#E11D48",
      "logo": "/src/assets/logos/wahda.png",
      "rankHistory": [3, 2, 1, 1, 2, ...]
    }
  ]
}
```

### شرح الحقول

| الحقل | المعنى | مثال |
|-------|--------|------|
| `name` | اسم النادي بالعربية | `"الوحدة"` |
| `abbr` | اختصار 2-4 أحرف يظهر عند غياب الشعار | `"WDH"` |
| `color` | لون خط المسار (HEX) | `"#E11D48"` |
| `logo` | مسار صورة الشعار | `"/src/assets/logos/wahda.png"` |
| `rankHistory` | ترتيب الفريق في كل جولة بالترتيب | `[3, 2, 1, 1, ...]` |

### قواعد مهمة في `rankHistory`

- عدد القيم يجب أن يساوي عدد الجولات في `matchweeks`
- القيم من **1** (الأول) إلى **16** (الأخير)
- مثال: إذا كان الفريق ثالثاً في الجولة 1 وأول في الجولة 2 → `[3, 1, ...]`
- **كل جولة يجب أن يكون كل رقم فريداً** (لا يمكن لفريقين أن يكونا في المركز نفسه)

## الخطوة 4 — استخدام عناصر التحكم

بعد تشغيل المشروع في المتصفح ستجد أزرار التحكم أسفل الرسم البياني:

| الزر | الوظيفة |
|------|---------|
| **تشغيل** | يبدأ تحريك الرسم البياني جولة بجولة |
| **إيقاف** | يوقف الحركة مؤقتاً |
| **⏮** | يرجع جولة واحدة للخلف |
| **⏭** | يتقدم جولة واحدة للأمام |
| **إعادة** | يرجع إلى الجولة الأولى |
| **بطيء / عادي / سريع** | يتحكم في سرعة الحركة |

يمكنك أيضاً **النقر على شريط التقدم** للانتقال مباشرة إلى أي جولة.

---

## الخطوة 5 — تصدير الفيديو

1. اضغط زر **تصدير فيديو**
2. انتظر حتى تكتمل عملية التسجيل (يظهر شريط تقدم بالنسبة المئوية)
3. عند الانتهاء سيُنزَّل الملف تلقائياً باسم: `syrian-league-race.webm`

> **ملاحظة:** تصدير الفيديو يعمل في المتصفحات الحديثة (Chrome، Edge، Firefox).
> لا يعمل في Safari على iOS.

---

## الخطوة 6 — بناء النسخة النهائية (اختياري)

لإنتاج ملفات جاهزة للنشر على الويب:

```bash
npm run build
```

ستجد الملفات الناتجة في مجلد `dist/`.

## الأسئلة الشائعة

**كيف أغيّر عدد الجولات؟**
عدّل مصفوفة `matchweeks` في `standings.json` وتأكد أن كل فريق لديه نفس عدد القيم في `rankHistory`.

**كيف أغيّر لون مسار فريق؟**
عدّل قيمة `color` في `standings.json` — أي لون HEX مقبول.

**الشعارات لا تظهر؟**
تأكد أن المسار في `logo` صحيح ويبدأ بـ `/src/assets/logos/` وأن الملف موجود فعلاً.

**الفيديو المُصدَّر فيه تقطع؟**
جرّب تغيير السرعة إلى "بطيء" قبل التصدير، أو فتح المشروع في Chrome.
